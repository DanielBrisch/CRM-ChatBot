Tipo de Correção 
(https://nimitz.atlassian.net/wiki/spaces/DPU/pages/2659877020/Tipos+de+Corre+o+de+Pedido):
B - Data base: Quando definido que realiza a correção por data base o sistema carrega os acréscimos e descontos definidos na tabela de preço e calcula a diferença de dias entre a data base e a prazo de pagamento, aplicando dessa forma um desconto ou acréscimo no valor do item.
N - Número de dias: Quando realiza o controle por número de dias é habilitado na tabela de preço filho o campo "Dias para Correção" sendo que possibilita o usuário a definir o número de dias para correção. Após isso o sistema verifica se a tabela de preço possui acréscimo e desconto nas formas de parcelamento, caso não tenha carregará os acrés/desc informado na tabela de preço e calcula a diferença de dias entre o prazo de pagamento e o número de dias da tabela de preço aplicando desta forma um desconto ou acréscimo no unitário do item.
P - Forma de Parc.: Quando realiza o controle por forma de parcelamento o sistema verifica se na forma de parcelamento da tabela de preço tem valores de acréscimo e descontos informados, caso contrário carrega os valores e adiciona o acréscimo ou desconto conforme o parcelamento.
Quando parametrizado que a correção é por “P - Forma de Parc.“ é possível adicionar na aba “Forma de Pagamento/Parcelamento“ quais são as possibilidades financeiras a serem utilizadas para a tabela de preço em específico. O sistema permite adicionar diversas formas de parcelamento para uma mesma forma de pagamento, sendo que é possível adicionar os percentuais de acréscimo a serem concedidos dos produtos conforme o parcelamento selecionado.
Validações:
“Não é possível incluir o item tal. A data de prazo para pagamento informada é maior que a data de prazo da tabela de preço”
Ocorre quando a “Tabela de Preço (Item x Preço)” possui uma Data prazo de pagamento, e essa é menor que a data informada no financeiro.
A validação ocorre ao incluir o item.
“As tabelas de Item x Preço vinculadas a tabela tal já expiraram! Por gentileza, verifique o cadastro das tabelas Item x Preço.”
Ocorre quando a “Tabela de Preço (Item x Preço)” possui data inicial e final da vigência, e essa já está vencida
A validação ocorre ao selecionar a tabela, ou tentar ir para as abas financeiro e produto

Campo “Data de Validade”
Valida na configuração do pedido (Agro3C » Pedidos » Configuração » Aba 5 - Controle de Datas)
“Altera Dt. Val. igual Dt. Pag”
Se “Sim”: conforme a data prazo para pagamento é alterada, automaticamente a data de validade também é alterada, ou seja, o default será o mesmo que o definido em data prazo para pagamento. Também obriga que a data de validade seja menor ou igual à data de prazo de pagamento.
Se “Não”: marca como default o definido na configuração “Data de Validade”.
“Dias máximo de Validade”
Quando definido um valor, não será possível alterar a data de validade para uma data maior que os dias definidos.
Funcionamento:

Dias entre a data base e o prazo de pagamento: Data base da tabela - Data prazo pagamento
% Total de Acréscimos: Acréscimos por dia x dia entre as datas
Preço do produto: Preço definido na tabela +% Total de acréscimos
Ex:
Acréscimos por dia: 10% ÷ 30 = 0,33
Dias entre a data base e o prazo de pagamento: 08/08/2023 -10/12/2023 = 124
% Total de Acréscimos: 0,33 x 124 =41,33%
Preço do produto: 300,00 +41,33% = 424,00
Se o prazo do pagamento é menor que a data base: Desconto

Aba Saldos
Validações do perfil do usuário
Seção “Saldos”
Configurações de quais saldos serão exibidos no produto
Seção “Estoque”
Seleciona Estoque
Se “Sim”:
Lista os saldos detalhados por local de estocagem, conforme os que foram selecionados para ficar visível.
O campo “Local de estoque” é exibido na tela de item, apenas se essa configuração estiver = Sim
Se “Não”:
Lista apenas os totais dos saldos configurados para ser exibido em tela
Considera nos totais do “Saldo Global”, apenas os estabelecimentos selecionados na configuração do pedido
Validação de saldo negativo (Agro3C » Pedido » Manutenção » Aba 2 - Itens):
“Saldo que não permite Negativo”
O saldo selecionado aqui é o que será validado no pedido
Quando configurado estabelecimentos (PEDCFGSALDONEG):
Será somado o saldo de todos os estabelecimentos selecionados para exibir no saldo.
Regra válida para exibição de saldos na tela consulta de produto
“Estabelecimentos”
Define quais estabelecimentos serão considerados no “Saldo Global” exibido no item
É somado os saldos de todos os estabelecimentos selecionados, se nenhum foi selecionado, a seção saldo global não é exibida
“Controla Autorização de Saldo?”
Se “Sim”: Quando o item selecionado está com saldo negativo no local de estocagem informado, mostra validação para solicitar liberação do pedido
Se “Não”: Quando o item selecionado está com saldo negativo no local de estocagem informado, não permite incluir o item
Saldos de Representantes
Quando configurado “Conf. a partir Reserva Repres”, na aba “Saldos” do item irá exibir a seção “Totais da Reserva do Representante” (Agro3C » Configurações » Configuração » Agro » Aba Pedidos)
Itens em Promoção
Validações no perfil do usuário “Destacar Itens em Promoção”
Se “Sim”: Ao adicionar itens, se algum produto tiver promoção na tabela de preço pai, irá exibir uma tela com os itens da promoção
Se “Não”: Não exibe promoções
Valida na Tabela de Preço - Item x Preço (Notas » Tabelas » Tabela de Preço - Item x Preço)
No item marcar “Promoção = Sim”, informar valor de promoção (normal e indexador) e validade da promoção
Controle de Cota
Definir na configuração do pedido, “Atualiza Cota do Item” (Agro3c » Pedidos » Configuração » Aba 2 - Itens)
Se “Sim”: ao adicionar o item, e informar uma quantidade maior que a cota disponível, ao salvar o item será validado e será alterada a quantidade automaticamente para o máximo disponível.
Se “Não”: Não valida cotas
Cadastrar uma cota (Agro3c » Pedidos » Controle de Cota)
Informar uma data inicial e final de vigência desta cota, se o controle será realizado por Data de Emissão do Pedido ou pela Data de Previsão de Entrega.
Controla cota por:
Grupo de Item: Quando selecionada esta opção é definido que a cota para o representante será por quantidade do Grupo do item.
Item: Quando selecionada esta opção é definido que a cota para o representante será por quantidade do item em específico.
Controla cota para: 
Representante: Quando selecionada esta opção, deve ser validado primeiramente o campo Saldo (Grid Representante), então verificar se os itens adicionados nos pedidos compõem o controle de cotas (Grid Item/Grupo com Controle de Cota).
Item/Grupo x Representante: Quando selecionada esta opção, deve ser validado a opção Controla Cota Por(Item ou Grupo), e então validar o campo Saldo localizado no GRID Item/Grupo com Controle de Cota se o mesmo não excede a cota para o representante.
Na grid, informar o representante, itens/grupos e a quantidade da cota.
Valida apenas cotas válidas e ativas
Para testes: usar configuração 1, tabela 192, item 5 (representante 1) ou 216 (representante 5)
Itens de Kit
Valida as configurações
No item (Agro3C » Estoque » Item » Manutenção)
“É Kit?” = Sim 
Definir itens que compõem o kit (Aba Composição - tabela ITEMCOMPOS)
Na “Tabela de Preço - Item x Preço”: 
Incluir na tabela os itens do kit e os que itens compõem o kit
Caso contrário irá validar “Os itens (X) não podem ser adicionados, pois não constam na tabela de preço”.
Na configuração do Pedido
“Iniciar Preços para Kit?” = Sim
No pedido, ao selecionar um item que é Kit, irá solicitar a quantidade, e ao confirmar irá incluir todos os itens que compõem aquele kit com a quantidade informada, fazendo a conversão de embalagem se houver.
Depois que o item kit é adicionado, não permite incluir novamente o mesmo item kit, ele não é mais listado na consulta de produto
Ao exclui um item da composição, os demais são excluídos
